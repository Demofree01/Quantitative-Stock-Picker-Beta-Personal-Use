# A 股周度量化选股器（个人使用 Beta）

> **AI 辅助生产声明**：本项目由用户提出业务规则与投资偏好，在 OpenClaw/Alice 的 AI 辅助下完成整理、改造、调试和文档编写。输出结果仅用于个人研究和人工决策，不构成投资建议。

## 1. 项目定位

这是一个每周运行一次的 A 股量化调仓计算器。

- 不自动交易。
- 不接券商 API。
- 不保证收益。
- 生成 Excel 报告后，由使用者人工判断是否买卖。

## 2. 主要功能

- 构建 A 股候选池。
- 拉取历史行情和最新价格。
- 计算趋势、动量、成交活跃、风险、估值等指标；财务质量字段当前无法稳定获取，已不参与评分。
- 生成候选买入 Top10、当前持仓检查、实际买入计划、调仓建议等表格。
- 支持读取 `holdings.csv` 作为手工基准持仓。
- 支持读取 `交割单.xlsx` 或旧版 `trades.csv`，自动滚动当前持仓与现金。
- 在 OpenClaw/headless 环境下可稳定运行。

## 3. 安装依赖

```bash
python3 -m pip install -r requirements.txt
```

如果在 OpenClaw 或 OneDrive/rclone 挂载目录下运行，建议使用 Tushare 独立入口：

```bash
bash run_tushare_once.sh
```

`run_tushare_once.sh` 会使用 `~/.openclaw/workspace/.venvs/stock_quant` 作为 Python 3.11 虚拟环境，并通过 `scripts/tushare_runner.py` 使用 Tushare 盘后数据。

## 4. 使用方式

### 4.1 准备持仓文件

创建 `holdings.csv`：

```csv
code,name,asset_type,shares,cost_price
605358,立昂微,STOCK,100,74.712
CASH,现金,CASH,5000,1
```

说明：

- 股票/ETF 行：`code,name,asset_type,shares,cost_price`
- 现金行：`CASH,现金,CASH,剩余现金金额,1`
- `asset_type` 可使用 `STOCK` / `ETF` / `CASH`。

### 4.2 准备交易流水（可选）

可以维护 `交割单.xlsx`，程序会先读取 `holdings.csv`，再根据交割单滚动出当前持仓和现金，并保存为 `自动持仓.csv` 方便核对。

如果没有 `交割单.xlsx`，程序仍兼容旧版 `trades.csv`。

### 4.3 运行程序

推荐前台运行一次：

```bash
bash run_tushare_once.sh
```

后台启动一次：

```bash
bash start_tushare_background.sh
```

说明：`weekly_quant.py` 保留核心算法；Tushare 数据获取、ETF 长线持仓处理、缓存与验收逻辑由 `scripts/tushare_runner.py` 适配。

### 4.4 查看结果

报告输出到：

```text
output/weekly_report_YYYYMMDD.xlsx
```

常用工作表：

- `系统摘要`
- `调仓建议`
- `实际买入计划`
- `候选买入Top10`
- `当前持仓检查`
- `全部评分`
- `失败列表`

## 5. 当前评分机制

### 5.1 基础过滤

在正式打分前，程序会剔除明显不适合的标的。主要规则来自 `config.yaml`：

- 排除 ST / 名称含 ST 的标的。
- 上市时间不少于 180 天。
- 股票最新价不低于 3 元。
- 近 20 日日均成交额不低于 8000 万元。
- 历史行情不少于 120 个交易日。
- 近 20 日涨幅不超过 45%。
- 近 60 日最大回撤不超过 35%。
- 若价格跌破 60 日均线且 20 日均线低于 60 日均线，则剔除。
- 如能获取流通市值，则优先执行 50 亿元流通市值过滤；如数据源缺失该字段，则程序容错跳过。
- 初始股票池按 Tushare 总市值排名选取：第 1-80 名 + 第 200-220 名 + 第 500-520 名；后续再做上市天数、成交额、涨幅、回撤、趋势等过滤。

### 5.2 综合评分公式

总分为四类因子的加权和：

```text
综合得分 = 趋势动量分 × 43.75%
        + 估值得分   × 23.75%
        + 流动性分   × 18.75%
        + 风险控制分 × 13.75%
```

其中：

```text
趋势动量分 = 动量分 × 55% + 趋势分 × 45%

动量分 = 近20日涨跌幅分位 × 30%
       + 近60日涨跌幅分位 × 45%
       + 近120日涨跌幅分位 × 25%

趋势分 = 最新价/60日均线分位 × 40%
       + 20日均线/60日均线分位 × 30%
       + 最新价/120日均线分位 × 30%

流动性分 = 近20日日均成交额分位 × 70%
         + 成交额放大倍数分位 × 30%

风险控制分 = 近60日年化波动率反向分位 × 50%
           + 近60日最大回撤反向分位 × 50%

财务质量分：已删除，不参与综合得分。原因是 ROE、经营现金流/净利润、扣非净利润同比当前无法稳定获取，继续参与会导致中性分/缺失值干扰排序。
```

估值得分：

- 当前使用 Tushare `daily_basic.pe_ttm`。
- 优先按行业 PE 基准做相对评分；行业无法映射时，才回退为候选池内 PE_TTM 反向分位。
- PE 缺失给容错低分，PE<=0 视为亏损/不可按便宜处理。
- PEG 当前禁用，不参与核心评分。

### 5.3 实际可获取的数据标注

当前稳定可获取并实际参与的字段：

- 股票代码、名称。
- 最新价（Tushare 盘后日线收盘价，不取盘中实时价）。
- 历史 K 线收盘价、最高价、最低价、成交额。
- 20/60/120 日均线。
- 近 20/60/120 日涨跌幅。
- 近 60 日年化波动率。
- 近 60 日最大回撤。
- 近 20/60 日日均成交额。
- 成交额放大倍数。
- 是否站上 60 日线、是否站上 120 日线、20 日线是否高于 60 日线。
- 当前持仓、成本价、持仓数量、现金余额（来自本地持仓/交割单）。

可获取但不完全稳定、因此采用容错处理的字段：

- 流通市值 / 总市值：来自 Tushare，用于市值排名和展示。
- PE_TTM：来自 Tushare `daily_basic.pe_ttm`，参与估值得分。
- 行业：来自 Tushare `stock_basic.industry`，用于行业 PE 基准映射；映射失败时估值分回退到全样本 PE_TTM 反向分位。
- **ROE、经营现金流/净利润、扣非净利润同比：当前无法稳定获取，已不参与评分；若偶尔获取到，仅作为排雷参考。**

当前禁用或不作为核心条件的字段：

- PEG：暂时禁用。
- 券商实时持仓：不接券商 API，仅使用本地文件。

## 6. 市场仓位判断

程序会根据候选池整体状态给市场活力分，满分 5 分：

- 站上 60 日线比例是否足够高。
- 站上 120 日线比例是否足够高。
- 近 20 日收益为正比例。
- 近 60 日收益为正比例。
- 成交额放大比例。

对应仓位：

- 弱市场：约 30%。
- 普通市场：约 60%。
- 强市场：约 85%。
- 活跃市场：约 95%。
